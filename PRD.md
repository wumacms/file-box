## 1. 产品概述

### 1.1 产品背景
在可视化建站平台中，图片及多媒体文件资源的组织与管理是核心基础。为了降低不同存储后端（OSS、Supabase Storage 等）的集成复杂度，并提供统一的、类似云盘的用户体验，特此开发「FileBox」。

### 1.2 产品定位
面向可视化建站平台的通用、插件化文件与媒体管理模块。

### 1.3 核心目标
1.  **统一管理**：支持多种存储后端，UI 无感知切换。
2.  **高效组织**：提供多级目录能力，支持批量操作（上传、移动、删除）。
3.  **无缝集成**：作为标准组件嵌入到 CMS 或 页面构建器中。
4.  **安全隔离**：确保用户资源相互独立。

## 3. 功能需求

### 3.1 用户认证系统 (Auth)
基于 Supabase Auth 实现基础账号能力。
*   **注册/登录**：支持邮箱/密码体系。
*   **状态保持**：支持 Session 自动刷新，维持登录状态。
*   **权限隔离**：所有资源操作必须基于当前登录用户的 UID。

### 3.2 存储系统 (Storage Provider)
核心设计目标是**插件化 (Plug-and-play)** 及 **多实例管理**。
*   **多存储桶支持**：支持用户配置和管理多个 OSS 存储桶，可无缝切换。
*   **接口标准化**：定义统一的 `StorageProvider` 接口（upload, delete, list, move）。
*   **当前实现**：阿里云 OSS。
*   **待扩展**：Supabase Storage, AWS S3, MinIO。

### 3.3 文件管理核心
*   **批量上传**：支持多文件同时上传，显示实时进度条。
*   **多选操作**：支持框选或勾选多文件进行批量删除、批量移动。
*   **多格式预览**：双击文件进入大图/全屏预览模式，支持图像、视频、PDF、文本、Markdown、CSV、HTML 等原生渲染和高亮展示，支持键盘左右键切换文件。
*   **信息详情**：悬停或点击可查看文件名、文件大小、存储路径、CDN URL。

### 3.4 目录管理 (Folder System)
*   **多级目录**：支持递归创建文件夹。
*   **导航路径**：面包屑导航，支持点击跳转至任意层级。
*   **安全删除**：若目录下存在文件，删除时需进行二次确认或禁止删除（视后端逻辑而定）。

### 3.5 文件/文件夹重命名 (Rename)
*   **即时重命名**：选中文件或文件夹后，支持弹出重命名对话框。
*   **后端同步**：重命名操作需同步更新存储路径（Copy + Delete 逻辑）。

### 3.6 文件夹删除 (Delete Folder)
*   **递归删除**：支持删除空文件夹及非空文件夹（需二次确认）。
*   **清理逻辑**：删除文件夹时，需批量清理其下的所有子文件。

### 3.7 文件移动与复制 (Move & Copy)
*   **跨目录移动**：选中文件后点击“移动”，弹出树状目录选择器，确认后更新存储路径。
*   **克隆功能**：支持将现有资源复制到指定目录。

### 3.8 全局搜索 (Search)
*   **文件名检索**：支持在当前目录或全站范围内按关键词搜索文件。

### 3.9 智能图片处理 (Image Processing) (针对图片文件)
*   **动态缩略图**：前端请求图片时可指定尺寸，后端（OSS）实时生成缩略图以节省流量。

### 3.10 存储统计
*   **容量监控**：展示当前用户已占用的总存储空间及文件总数。

### 3.11 多存储桶管理 (Multi-Bucket Management)
*   **独立设置页**：废弃原有的弹窗式设置，改为独立的系统设置页面（路由跳转），提供更充足的配置空间。
*   **多存储桶配置**：允许用户添加、编辑、删除多个 OSS 存储桶配置（包括 Bucket 名称、区域、AccessKey 等）。
*   **侧边栏联动**：侧边栏优先展示用户的所有存储桶列表。点击存储桶后，右侧动态加载并展示该存储桶内的文件与目录，同时侧边栏展开该存储桶的目录树。

## 4. UI/UX 设计要求

### 4.1 核心布局
采用经典的「左侧树 + 右侧网格」布局：
*   **左侧导航栏**：顶层展示用户配置的**多个存储桶列表**，展开后展示对应存储桶的文件夹树状结构。
*   **顶部工具栏**：[上传]、[新建文件夹]、[批量移动]、[批量删除]、[搜索(规划中)]。
*   **中心展示区**：文件瀑布流或网格，支持响应式布局（4-6列），根据文件类型展示相应缩略图或图标。

### 4.2 交互细节
*   **Hover 态**：文件卡片在悬停时显示勾选框和快捷操作图标。
*   **空状态**：当目录下无内容时，展示精美的占位图及引导操作。
*   **反馈机制**：操作成功/失败均需有明显的 Toast 提示。

## 5. 技术架构设计

### 5.1 系统分层
1.  **UI 层**：Vue 3 + TailwindCSS + Element Plus。
2.  **业务逻辑层**：处理文件选择状态、目录切换逻辑。
3.  **抽象适配层**：定义 `IStorageProvider`。
4.  **驱动实现层**：`OSSProvider`, `SupabaseProvider` 等。

### 5.2 数据模型 (示例)
**文件 (FileItem)**
```typescript
interface IFileItem {
  id: string;      // 唯一标识
  name: string;    // 文件名
  path: string;    // 存储全路径
  url: string;     // 可访问链接
  size: number;    // 字节数
  type: string;    // mime type
  updatedAt: Date;
}
```

**目录 (Folder)**
```typescript
interface IFolder {
  name: string;
  path: string;    // 父路径/当前名/
  children?: IFolder[];
}
```

## 6. 安全与非功能需求

### 6.1 安全策略
*   **凭证安全**：前端严禁存放 AccessKeySecret。OSS 必须使用 STS 临时凭证或后端签名。
*   **存储隔离**：桶(Bucket)内路径强制以 `userId/` 为根前缀。
*   **访问控制**：启用 Supabase RLS，防止垂直或水平越权。

### 6.2 性能优化
*   **缩略图优化**：对于图片格式，列表展示应使用 OSS/CDN 的缩略图参数（如 `?x-oss-process=image/resize,w_200`）以减少带宽和渲染压力。
*   **懒加载**：网格组件采用 Intersection Observer 进行延迟渲染。

## 7. 迭代规划

### 第一阶段 (V1.0) - 核心闭环
*   集成 Supabase Auth。
*   实现 OSS 存储对接。
*   完成文件上传、多选删除、多格式文件预览、多级目录创建。
*   支持文件/文件夹重命名、文件夹递归删除。
*   支持跨目录文件移动功能。
*   支持全局文件名快速搜索。

### 第二阶段 (V2.0) - 效率提升
*   支持拖拽上传及拖拽移动文件夹。
*   基础文件处理（图片裁剪、视频封面截取）。

### 第三阶段 (V3.0) - 系统增强
*   动态文件处理（水印、转码）。
*   存储容量统计与超限提醒。
*   多存储后端动态切换 UI。

---

## 8. 验收指标 (KPI)
*   **成功率**：图片上传失败率 < 0.5%。
*   **加载速度**：首页及首屏网格加载时间 < 1.5s (标准网络环境下)。
*   **易用性**：核心任务（上传并插入页面）的操作路径不超过 3 步。
